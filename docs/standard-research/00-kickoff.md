# Standard Basic Print Research

## Purpose

This research track defines and verifies Luceon's `basic_print` standard: the lowest acceptable printable textbook edition generated by the Standard stage.

The goal is not to optimize the current Standard output cosmetically. The goal is to define a source-faithful, evidence-backed, profile-driven standard that can decide whether a generated textbook package is actually usable for printing.

## Project Thesis

Standard must be a semantic printable compiler, not a prettier Clean renderer.

```text
Clean package + Raw evidence + source PDF
  -> StandardDocument
  -> profile-aware layout intent
  -> neutral printable HTML/PDF
  -> acceptance reports
```

Standard succeeds only when the generated package preserves source content, keeps learning context intact, and renders into a plain printable edition that students and teachers can use without opening the original PDF.

## Scope

This track owns:

- quality levels for Standard output;
- the `basic_print` definition;
- profile contracts and normalization rules;
- regression corpus policy;
- acceptance gates for source fidelity, semantic context, print usability, and auditability;
- implementation sequencing for turning current Standard MVP output into verified `basic_print` output.

This track does not own:

- decorative theme systems;
- brand design or polished publication styling;
- rewriting or expanding source content;
- manual editing of Standard output as a substitute for compiler/profile fixes.

## Guiding Rules

1. Do not call generated files usable merely because they exist.
2. Do not call a package printable merely because Chrome generated a PDF.
3. Do not call Clean success Standard success.
4. Keep source case, Standard run, and golden output as separate objects.
5. Prefer `review` over false `pass`.
6. Treat layout as semantic when it affects learning, question answering, image interpretation, table reading, or formula correctness.
7. Keep theme and polish outside the `basic_print` hard gates.
